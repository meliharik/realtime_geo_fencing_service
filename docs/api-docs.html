API Reference

Complete documentation for the Real-Time Geo-Fencing API

Interactive Documentation: For interactive API testing with "Try it out" functionality, run the application locally and visit http://localhost:8080/swagger-ui.html

REST API Endpoints

POST /api/geofencing/check

Check if a GPS coordinate violates any no-parking zones. Uses Redis cache for fast lookups (~0.1-0.5ms) with PostGIS fallback (~5-10ms).

Request Body

{
  "scooterId": "SC-001",
  "latitude": 37.7800,
  "longitude": -122.4150,
  "timestamp": "2024-01-01T12:00:00Z"
}

Response (No Violation)

{
  "status": "OK",
  "message": "No violations detected",
  "scooterId": "SC-001",
  "location": {
    "latitude": 37.78,
    "longitude": -122.415
  }
}

Response (Violation Detected)

{
  "status": "VIOLATION",
  "message": "Zone violation detected!",
  "scooterId": "SC-001",
  "violations": [{
    "violationId": "VIO-123",
    "scooterId": "SC-001",
    "zoneId": 1,
    "zoneName": "Downtown SF",
    "latitude": 37.78,
    "longitude": -122.415,
    "timestamp": "2024-01-01T12:00:00Z",
    "severity": "HIGH",
    "distanceFromBoundary": 0.0
  }]
}

cURL Example

curl -X POST http://localhost:8080/api/geofencing/check \
  -H "Content-Type: application/json" \
  -d '{
    "scooterId": "SC-001",
    "latitude": 37.7800,
    "longitude": -122.4150,
    "timestamp": "2024-01-01T12:00:00Z"
  }'
GET /api/geofencing/check-quick

Quick violation check using query parameters instead of JSON body. Easier for browser testing.

Query Parameters

Parameter Type Required Example
scooterId String Yes SC-001
lat Double Yes 37.7800
lon Double Yes -122.4150

Example

GET http://localhost:8080/api/geofencing/check-quick?scooterId=SC-001&lat=37.7800&lon=-122.4150
GET /api/geofencing/zones

Get all active no-parking zones from the database.

Response

[{
  "id": 1,
  "name": "Downtown SF Test Zone",
  "description": "No parking zone in downtown SF",
  "active": true,
  "severity": "HIGH",
  "createdAt": "2024-01-01T00:00:00",
  "updatedAt": "2024-01-01T00:00:00"
}]
GET /api/geofencing/cache/stats

Get Redis cache statistics and health metrics.

Response

{
  "cachedZoneCount": 5,
  "databaseZoneCount": 5,
  "cacheHealthy": true,
  "cacheHitRate": "99.5%"
}
POST /api/geofencing/cache/refresh

Manually refresh the Redis cache from the database.

Response

{
  "status": "SUCCESS",
  "message": "Cache refreshed",
  "zonesRefreshed": 5
}
GET /api/geofencing/violations/{scooterId}

Get violation history for a specific scooter.

Example

GET http://localhost:8080/api/geofencing/violations/SC-001

WebSocket API

Connect to the WebSocket endpoint for real-time GPS streaming and violation alerts.

Connection URL

ws://localhost:8080/ws/gps-stream

Available Topics

Destination Type Description
/app/gps Send Send single GPS event
/app/gps/batch Send Send batch GPS events
/app/ping Send Health check ping
/topic/alerts Subscribe Receive violation alerts
/topic/status Subscribe Receive system status updates
/user/queue/reply Subscribe Receive personal responses

JavaScript Example

// Connect to WebSocket
const socket = new SockJS('http://localhost:8080/ws/gps-stream');
const stompClient = new StompJs.Client({
    webSocketFactory: () => socket
});

// Setup connection handler
stompClient.onConnect = () => {
    console.log('Connected');

    // Subscribe to violation alerts
    stompClient.subscribe('/topic/alerts', (message) => {
        const alert = JSON.parse(message.body);
        console.log('Violation:', alert);
    });
};

// Connect
stompClient.activate();

// Send GPS event
stompClient.publish({
    destination: '/app/gps',
    body: JSON.stringify({
        scooterId: 'SC-001',
        latitude: 37.7800,
        longitude: -122.4150,
        timestamp: new Date().toISOString()
    })
});
